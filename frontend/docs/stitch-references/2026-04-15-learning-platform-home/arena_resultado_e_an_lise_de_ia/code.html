The Reasoning Error

You assumed the pointer ptr remained valid after the push() operation. While a Segfault is a possible manifestation, it is not guaranteed.

Logic Flaw Detected

You mistook a specific hardware outcome (Segfault) for the abstract language rule (Undefined Behavior).

Architectural Explanation

When data.push(4) is called, the vector may exceed its current capacity. This triggers a reallocation: the vector allocates a larger block of memory elsewhere, copies the elements, and frees the old block. Your raw pointer ptr is now "dangling"—it points to freed memory. Reading it is Undefined Behavior (UB).
